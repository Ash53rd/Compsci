Regex Tutorial: Email Validation

Introduction

Welcome to this regex tutorial! As a web development student, understanding regular expressions (regex) is crucial for pattern matching and input validation. This tutorial will break down a regex pattern used for email validation, explaining each component to help you understand how it works.

Summary

This tutorial will explain the following regex pattern used for email validation: 

^([a-z0-9_\.-]+)@([\da-z\.-]+)\.([a-z\.]{2,6})$

This regex pattern is designed to match valid email addresses by breaking them down into three main parts: the local part (before the @), the domain name, and the top-level domain.

Regex Components

Anchors

  • ^: Marks the start of the string
  • $: Marks the end of the string

These anchors ensure that the regex matches the entire string from start to finish, preventing partial matches.

Quantifiers

  • +: Matches one or more of the preceding token
  • {2,6}: Matches between 2 and 6 of the preceding token

In our regex, + is used for the local part and domain name, allowing them to be of any length greater than zero. {2,6} is used for the top-level domain, restricting it to between 2 and 6 characters.
